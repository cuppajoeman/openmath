Path Tracing

Path tracing is a Monte Carlo method for estimating camera-ray radiance by sampling light transport paths through a rendering scene. This page builds on the radiometric quantities and surface-scattering definitions from radiometry.

Rendering Scene
A rendering scene is the collection of surfaces, light sources, and material scattering functions used to determine the radiance reaching a camera.

Path Notation

Directed Point Pair
If \(p,q\in\mathbb R^3\) and \(p\neq q\), then \[ p\to q \] denotes the direction from \(p\) toward \(q\). As a unit vector, this direction is \[ \omega_{p\to q} = \frac{q-p}{\left\|q-p\right\|}. \]

The notation \(p\to q\) is not a new kind of point. It is shorthand for asking about light traveling from the location \(p\) toward the location \(q\).

Radiance Along a Directed Point Pair
If \(p,q\in\mathbb R^3\) and \(p\neq q\), then \[ \operatorname{radiance}(p\to q) \] means the radiance at \(p\) in the direction from \(p\) toward \(q\): \[ \operatorname{radiance}(p\to q) = \operatorname{radiance}\left(p,\omega_{p\to q}\right). \] When \(p\) lies on a surface and the light is leaving \(p\) toward \(q\), this is an exitant radiance value.
Camera Ray Radiance Estimate
Suppose a camera ray starts at \(p_0\) and first intersects the scene at \(p_1\). The quantity \[ \operatorname{radiance}(p_1\to p_0) \] is the radiance leaving the first intersection point \(p_1\) back toward the camera point \(p_0\). Path tracing estimates this value by summing contributions from valid light transport paths that begin at \(p_0\) and then visit \(p_1\).
Pixel Sample
A pixel sample is one sampled point within a pixel's image-plane region. A renderer usually estimates a pixel value by tracing one camera ray for each pixel sample and averaging the returned radiance estimates.
Camera Ray
A camera ray is a ray that starts at the camera and travels through a point on the image plane or pixel sample. A path tracer estimates the incident radiance carried back along this ray.
Ray-Scene Intersection
A ray-scene intersection is the first point where a ray meets a surface in a rendering scene. If no such point exists, the ray escapes the scene.
Visibility Between Points
Two points in or around a rendering scene are visible to each other if the open line segment between them does not intersect any blocking surface in the scene. The visibility factor \(V(p,q)\) is \(1\) when \(p\) and \(q\) are visible to each other and \(0\) otherwise.
Light Sample
A light sample at a surface point \(p\) is a randomly chosen point or direction on a light source, together with the sampled incident radiance and the density used to choose that sample.
Direct Lighting Estimator
A direct lighting estimator estimates the one-bounce light arriving at a surface point by sampling a light source. For a light sample from direction \(\omega_i\) with density \(p_l(\omega_i)\), one sample contributes \[ \frac{ \operatorname{bsdf}\left(p,\omega_i,\omega_o\right) \operatorname{radiance\_in}\left(p,\omega_i\right) \left|n\cdot\omega_i\right| V }{ p_l\left(\omega_i\right) }, \] where \(\operatorname{bsdf}\) is the BSDF, \(\operatorname{radiance\_in}\) is incident radiance, \(n\) is the surface normal, and \(V\) is the visibility factor.
BSDF Sample
A BSDF sample at a surface point chooses a new direction \(\omega_i\) according to a density \(p_{\mathrm{dir}}\) on the unit sphere \(\mathbb{S}^2\), measured with respect to solid angle. Thus \(p_{\mathrm{dir}}\left(\omega_i\right)\) is the density assigned to sampling the particular direction \(\omega_i\). The sample also returns \(\operatorname{bsdf}\left(p,\omega_i,\omega_o\right)\), the value of the BSDF for the sampled incoming direction and the outgoing direction. This is an importance sampling step for the BSDF scattering equation.

Choosing a direction according to \(p_{\mathrm{dir}}\) means that the renderer uses random numbers to produce a direction-valued random variable whose density is \(p_{\mathrm{dir}}\). In one dimension, a standard way to do this is the inversion method: start with a uniform random number and transform it so the output has the desired distribution. Direction sampling uses the same idea, but the output is a point \(\omega\) on \(\mathbb{S}^2\) instead of a real number. By the definition of density, having density \(p_{\mathrm{dir}}\) means that for any set of directions \(A\subseteq\mathbb{S}^2\), the probability that the sampled direction lands in \(A\) is \(\int_A p_{\mathrm{dir}}\left(\omega\right)\,d\omega\).

Operationally, a renderer implements this with a sampling procedure: a function that takes one or more uniform random numbers and returns a direction. The function is designed so that the preimage of a direction patch \(A\) occupies exactly \(\int_A p_{\mathrm{dir}}\left(\omega\right)\,d\omega\) worth of area in the uniform random-number domain. Larger-density direction patches are assigned larger regions of the input random-number square, so they occur more often. The path tracer later divides by \(p_{\mathrm{dir}}\left(\omega_i\right)\), so sampling some directions more frequently changes how noisy the estimate is, not which radiance integral is being estimated.

Path Throughput
The path throughput is the accumulated multiplicative estimator weight of a sampled light path. If a path has already scattered through directions \(\omega_1,\ldots,\omega_k\), then its throughput has the form \[ \beta = \prod_{j=1}^{k} \frac{ \operatorname{bsdf}\left(p_j,\omega_j,\omega_{j-1}\right) \left|n_j\cdot\omega_j\right| }{ p_{\mathrm{dir}}\left(\omega_j\right) }, \] where each denominator is the density used when sampling the corresponding direction. The throughput is the reason a later light contribution is multiplied by all earlier scattering weights.
Path Throughput from Importance Sampling
The path throughput factor comes from applying the Monte Carlo estimator to the BSDF scattering equation. At a surface point \(p\), with outgoing direction \(\omega_o\), surface normal \(n\), and sampled incoming direction \(\omega_i\) drawn from density \(p_{\mathrm{dir}}\), the one-sample estimator for the scattered incident radiance term is \[ \frac{ \operatorname{bsdf}\left(p,\omega_i,\omega_o\right) \operatorname{radiance\_in}\left(p,\omega_i\right) \left|n\cdot\omega_i\right| }{ p_{\mathrm{dir}}\left(\omega_i\right) }. \] Therefore the estimate of whatever radiance is found by continuing the path in direction \(\omega_i\) must be multiplied by \[ \frac{ \operatorname{bsdf}\left(p,\omega_i,\omega_o\right) \left|n\cdot\omega_i\right| }{ p_{\mathrm{dir}}\left(\omega_i\right) }. \] Repeating this at each bounce multiplies these factors together, which gives the path throughput \(\beta\).
In the BSDF scattering equation, the scattered part of \(\operatorname{radiance\_out}\left(p,\omega_o\right)\) is an integral over incoming directions: \[ \int_{\mathbb{S}^2} \operatorname{bsdf}\left(p,\omega_i,\omega_o\right) \operatorname{radiance\_in}\left(p,\omega_i\right) \left|n\cdot\omega_i\right| d\omega_i. \] If the renderer samples \(\omega_i\) from density \(p_{\mathrm{dir}}\), the Monte Carlo estimator replaces the integrand by the sampled integrand divided by the sampling density: \[ \frac{ \operatorname{bsdf}\left(p,\omega_i,\omega_o\right) \operatorname{radiance\_in}\left(p,\omega_i\right) \left|n\cdot\omega_i\right| }{ p_{\mathrm{dir}}\left(\omega_i\right) }. \] The remaining term \(\operatorname{radiance\_in}\left(p,\omega_i\right)\) is the radiance arriving from the next ray direction, so a recursive path tracer estimates it by continuing the path. Thus the continuation estimate is multiplied by the non-\(\operatorname{radiance\_in}\) part of the sampled term. After \(k\) sampled scattering events, these multiplicative factors form the product in the definition of path throughput.
Light Transport Path
Let \(S\) be a rendering scene, and let \(p_0\in\mathbb R^3\) be an initial point. The set of valid light transport paths from \(p_0\) through \(S\) is denoted \[ \operatorname{LTP}\left(p_0,S\right). \] An element of \(\operatorname{LTP}\left(p_0,S\right)\) is a tuple \[ \left(p_1,\ldots,p_k\right) \] such that each \(p_i\) lies on a surface in \(S\), and each adjacent pair \[ p_i,\ p_{i+1} \] is visible in \(S\), where the index \(i\) ranges from \(0\) to \(k-1\). Thus the open line segment from \(p_i\) to \(p_{i+1}\) hits no other surface in the scene before reaching \(p_{i+1}\). The initial point \(p_0\) is not part of the tuple; it is supplied as an argument of \(\operatorname{LTP}\left(p_0,S\right)\).
Fixed-Length Light Transport Paths
Let \(S\) be a rendering scene, let \(p_0\in\mathbb R^3\), and let \(k\in\mathbb N_1\). The set \[ \operatorname{LTP}_k\left(p_0,S\right) \] is the subset of valid light transport paths from \(p_0\) through \(S\) whose elements are \(k\)-tuples: \[ \operatorname{LTP}_k\left(p_0,S\right) = \left\{ \left(p_1,\ldots,p_k\right)\in\operatorname{LTP}\left(p_0,S\right) \right\}. \] Thus \(\operatorname{LTP}_1\left(p_0,S\right)\) contains one-vertex paths, \(\operatorname{LTP}_2\left(p_0,S\right)\) contains two-vertex paths, and so on.
Path Radiance Contribution
Let \(S\) be a rendering scene, and let \[ \left(p_1,\ldots,p_k\right)\in\operatorname{LTP}\left(p_0,S\right). \] The notation \[ \operatorname{path\_rad}\left(p_0;\,p_1,\ldots,p_k\right) \] denotes the radiance contribution assigned to that path. The exact formula depends on the path's emitted radiance, BSDF factors, geometric factors, and visibility. The sampling density appears later when a Monte Carlo estimator chooses which paths to sample. At this stage, \(\operatorname{path\_rad}\left(p_0;\,p_1,\ldots,p_k\right)\) should be read as one path's contribution to the camera-ray radiance value.
Path Tracing Overview Equation
Let \(S\) be a rendering scene. If a camera ray starts at \(p_0\) and first hits the scene at \(p_1\), then the path tracing goal can be written schematically as \[ \operatorname{radiance}\left(p_1\to p_0\right) = \sum_{k=1}^{\infty} \sum_{\left(p_1,\ldots,p_k\right)\in\operatorname{LTP}_k\left(p_0,S\right)} \operatorname{path\_rad}\left(p_0;\,p_1,\ldots,p_k\right). \] The left side is the radiance leaving the first intersection point back toward the camera. The right side says that this radiance is assembled from path radiance contributions of valid light transport paths. This equation is schematic: later, sums over continuously many surface points must be interpreted as integrals.
Recursive Path Depth
The recursive path depth of a sampled path is the number of surface scattering events that have already been used to extend the path away from the camera.
Maximum Path Depth
The maximum path depth is the largest recursive path depth that a path tracer allows before it stops extending the current light transport path.
Simple Path Tracer
A simple path tracer is a Monte Carlo estimator for camera-ray radiance. It repeatedly intersects a ray with the scene, adds emitted and directly sampled light scaled by the current path throughput, samples a new direction using \(\operatorname{bsdf}\) from the surface BSDF, updates the throughput, and continues until the path ends or reaches its maximum path depth.
Simple Path Tracing Loop
Given a camera ray for one pixel sample, initialize accumulated radiance \(L=0\), path throughput \(\beta=1\), and recursive path depth \(d=0\). The current ray always points away from the previous path vertex. At each intersection point \(p\), let \(n\) be the surface normal at \(p\), and let \(\omega_o\in\mathbb S^2\) point from \(p\) back along the current ray toward the previous path vertex. Then repeat:
  1. Find the next ray-scene intersection. If the ray escapes, add any visible environment emitted radiance scaled by \(\beta\), and stop.
  2. If the intersected surface emits light, add \(\beta \operatorname{emitted\_radiance}\left(p,\omega_o\right)\) to \(L\), where \(\operatorname{emitted\_radiance}\left(p,\omega_o\right)\) is the emitted radiance at the intersection point \(p\) in the outgoing direction \(\omega_o\) back toward the previous path vertex.
  3. If direct light sampling is enabled, take a light sample and add its direct lighting estimator contribution scaled by \(\beta\) to \(L\).
  4. If \(d\) has reached the maximum path depth, stop.
  5. Take a BSDF sample to choose the next direction \(\omega_i\).
  6. Update \[ \beta \leftarrow \beta \frac{ \operatorname{bsdf}\left(p,\omega_i,\omega_o\right) \left|n\cdot\omega_i\right| }{ p_{\mathrm{dir}}\left(\omega_i\right) }. \]
  7. Spawn the next ray in direction \(\omega_i\), set \(d\leftarrow d+1\), and continue. At the next intersection, the new \(\omega_o\) is the direction back toward the current point \(p\).
